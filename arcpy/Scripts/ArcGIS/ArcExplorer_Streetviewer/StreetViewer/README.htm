ArcGIS Explorer Custom Template

Readme File

Below you can find some information on setting up and using this template.

Coding your custom task

The template code which has been added to your new project defines a functioning custom task. If this is the first time you have used the template, you should familiarize yourself with the contents of the project.

  • Open the code view of the CustomTaskUI class; this class provides information to the ArcGIS Explorer application about the custom task.
    The existing code in this class links together the CustomTaskUI and CustomTaskControl classes, and provides icons and a context menu for results arising from the task execution.
  • Open the Designer View of the CustomTaskControl class; this class provides the user interface for the task, by inheriting from the System.Windows.Forms.UserControl class. In this class you can add code to work with the E2API namespace and extract information which you wish to display to the user or to pass to the CustomTask.
    The existing code in this class allows a user to click on a button and then click on the current map view in explorer. Clicking the Find button will add a parameter containing the location information to a ParameterSet, and then start the CustomTask executing by calling ProcessTask, passing in the ParameterSet of information collected from the user click.
  • Open the code view of the Task file, called CustomTask. In the Execute method you can add code to perform the main function of your custom task. 
    The existing code in this class takes the input coordinates and Geometry and creates a PlaceResult from this information.

Testing your custom task

In order to use your custom task, you need to build it and then add it to a map.

Build the Task DLL

  1. Build your custom task project.

  2. In Visual Studio 2005, open the solution and build it.

    NOTE: see the 'Building and Using the samples' topic in the help system for more information about how the task created by this project will be automatically set up ready for you to use in ArcGIS Explorer by building the project, using a project post-build event and the CopyToTasks tool. Having this post build step ensures two things:

    • The new task is ready to use as soon as you build it.
    • Each time you rebuild your project, the new version will be copied automatically to the Tasks folder so that you can test it straight away.

    Note that if you are using Visual Studio Express editions, post build events cannot be edited using the Visual Studio Express edition user interface. For more information, see the Using Visual Studio topic in the ArcGIS Explorer Developer Help.

Use the Task DLL

  1. Open ArcGIS Explorer, and open the map that you wish to add your task to.
  2. Choose Tools, Manage Tasks.
  3. As the post-build step of the template has automatically set up the task for you to test, you should now see the name of your new task listed in the left-hand pane.
  4. Select your task and click the arrow in the center to add the task to the current map.
  5. Click OK to dismiss the dialog.

You should now see your custom task listed in the task center.

An alternative method is to open a map, and then use the Add Content option in the File menu to again browse to an NMF file which contains your task information. See the help topics about NMF files and Tasks for more information.

Click on the button with the 'flash' icon and then click somewhere on the map. You should see a result added to the results window, and a blue pushpin indicating the location where you clicked the map.

Deploying your custom task

At the point when you are ready to deploy your custom task to others, you have a number of options available.

  • Place the custom Task at a server location which will be available to your users, and edit the DownloadLocation element in the .NMF file to reflect this location. Then add the custom Task to a Map, and publish the Map to the Server. Users can then open the Map from the server, and the Task DLL will be downloaded to their machine.
  • Place the custom task on a server and edit the DownloadLocation of the .NMF file accordingly. Then add the task to a map, and save the map locally. You can then give this map to another user, and the DownloadLocation saved in to that map will be used to attempt to download the task DLL on the other users machine when they open the map.
  • You can give your custom task directly to another user; they must follow a similar procedure to run the task as you used above; put the task in the Tasks directory and add it to the Map using the .NMF file.

NOTE: The downloading of custom tasks is handled by the NMF file in all cases; if you save a map with your custom task included, the DownloadLocation from the .NMF file you chose is also included in the map, and will be used to attempt to download the custom task when other users open that map. Ensure the .NMF file includes the correct information for the location, name and version of your custom project binary file, and that location is accessible by the user.

Refer to the Adding and Deploying Custom Tasks section of the ArcGIS Explorer Developer Help system for more information.
